1 · Dataverse como knowledge

El agente puede responder sobre registros reales, no solo sobre documentos.

Dataverse se puede añadir como fuente de conocimiento en Copilot Studio para fundamentar respuestas con datos de tablas.

Aplicación típica: ideal para preguntas sobre clientes, incidencias, activos, solicitudes, estados, histórico y relaciones entre entidades, sin crear primero una API específica.
Desbloquea
  • Consultas en lenguaje natural sobre tablas.
  • Uso de relaciones y registros relacionados.
  • Sinónimos y glosario para nombres técnicos.
  • Seguridad basada en permisos del usuario.
Límites
  • Requiere autenticación Microsoft.
  • Necesita Dataverse Search.
  • El límite documentado es de fuentes y tablas por agente.
  • La calidad depende de nombres, descripciones y relaciones.

2 · App MCP · Capacidad prioritaria

La Model-driven app puede publicarse como experiencia operativa en Microsoft 365 Copilot.

Con App MCP, Power Apps crea un servidor MCP asociado a la aplicación y un agente declarativo que expone vistas, formularios y herramientas a Copilot.

01

Consultar

Copilot puede recuperar filas de las tablas incluidas en la app y responder con contexto.

02

Visualizar

Puede mostrar grids, formularios y widgets MCP en la conversación.

03

Operar

Puede guiar creación y edición de registros usando la experiencia de la app.

04

Extender

Puede añadirse lógica propia mediante herramientas MCP personalizadas.

La app deja de ser solo una pantalla: pasa a ser una herramienta que Copilot puede invocar.

2A · App MCP · Flujo de uso

Del correo o reunión al registro en Dataverse, sin obligar al usuario a navegar por la app.

Entrada

Microsoft 365 Copilot

Usuario pregunta desde Chat, Teams, Outlook o una experiencia de Copilot habilitada.

Orquestación

Agente declarativo + App MCP

El agente decide qué vista, formulario o herramienta de la app debe invocar.

Acción

Model-driven app / Dataverse

Se consultan, muestran o preparan operaciones sobre registros respetando seguridad.

Ejemplo 1

“Muéstrame las solicitudes críticas abiertas y abre las que no tienen responsable.”

Ejemplo 2

“Crea un borrador de incidencia con este correo y déjalo listo para revisión.”

Ejemplo 3

“Resume los activos de este cliente y genera una tabla de riesgos.”

Decisión recomendada: pilotar App MCP primero sobre una app acotada, con pocas tablas críticas, formularios claros y un proceso de negocio concreto.

4 · Power Apps MCP + agent feed

Los agentes pueden trabajar; el usuario supervisa dentro de la Model-driven.

Power Apps MCP añade un patrón de “human-in-the-loop”: el agente no desaparece en segundo plano; deja tareas, solicitudes y revisiones dentro de la app.

Valor principal: procesos con datos sensibles o incompletos pueden beneficiarse de IA sin perder control operacional.
log_for_review

Registrar acciones completadas para revisión.

request_assistance

Pedir intervención humana cuando el agente no debe decidir solo.

invoke_data_entry

Convertir documentos, correos o texto en sugerencias de formulario.

5.9 · Agent Response

Respuestas de agentes embebidas en el formulario, con contexto del registro.

El componente Agent Response permite ejecutar un topic de Copilot Studio desde un formulario Model-driven y mostrar la respuesta directamente en la experiencia del usuario.

Contexto implícito

App + página + registro

El agente recibe contexto del entorno donde está el usuario.

Salida rica

Markdown / Adaptive Cards

Permite respuestas estructuradas, recomendaciones, paneles y acciones.

Uso directo

Bloques de inteligencia

Riesgo, resumen, siguiente acción, cumplimiento, documentación faltante.

Patrón recomendado: empezar con un componente “Resumen + recomendación” en la entidad más crítica, sin automatizar escritura hasta validar calidad.

6 · Dataverse Intelligence / Semantic Model

La calidad del modelo de datos empieza a ser calidad de IA.

Dataverse semantic model da a Copilot y a los agentes una comprensión empresarial compartida: no solo nombres técnicos de columnas, sino significado, relaciones, vistas, formularios y señales semánticas.

Señales
  • Tablas y columnas.
  • Relaciones.
  • Vistas públicas.
  • Formularios.
  • Descripciones.
Gobierno
  • Excluir ruido.
  • Afinar señales.
  • Documentar términos.
  • Validar respuestas.
  • Preparar datos sensibles.
Impacto
  • Mejores respuestas.
  • Menos ambigüedad.
  • Mejor recuperación.
  • Mayor confianza.
  • Menos prompts correctivos.

7 · Business skills

Las reglas del negocio dejan de vivir repetidas en cada prompt.

Business skills permite documentar procesos, criterios, pasos y políticas para que distintos agentes puedan descubrir y aplicar una misma lógica empresarial.

Aplicación típica: útil para procesos donde la decisión no depende solo del dato, sino de cómo la organización interpreta el dato.

Ejemplo

“Para clasificar una solicitud crítica, evalúa impacto, urgencia, cliente afectado, SLA y dependencias operativas.”

Uso

Un agente consulta la skill cuando necesita aplicar una política, no solo responder con datos.

Gobierno

Se gestionan como activos reutilizables y transportables entre entornos mediante soluciones.

Arquitectura propuesta

Una evolución por capas, no una reconstrucción de la aplicación.

Canal
Microsoft 365 Copilot, Teams, Outlook y la propia Model-driven app.
Agente
Agente declarativo generado por App MCP + agente de Copilot Studio para lógica específica.
Herramientas
App MCP para vistas/formularios, Dataverse MCP para datos/operaciones y herramientas custom para sistemas externos.
Supervisión
Power Apps MCP + agent feed para revisión humana, excepciones y entrada de datos asistida.
Semántica
Dataverse Intelligence, semantic model y business skills para que el agente entienda el lenguaje del negocio.
Core
Model-driven app existente, Dataverse, seguridad por roles, vistas, formularios, relaciones y auditoría.

De valor visible a operación agéntica controlada.

  • Limpiar nombres, vistas y formularios.
  • Definir glosario mínimo.

  • Power Apps MCP + agent feed.
  • Business skills.
  • Herramientas MCP custom.
  • Métricas de uso, precisión y ahorro.
Criterio de éxito: no medir solo “respuestas correctas”; medir reducción de navegación, tiempo de cumplimentación, calidad de datos y trazabilidad de decisiones.
90D
Consideraciones y límites

Antes de venderlo como productivo, conviene acotar expectativas.

Disponibilidad
  • Varias capacidades están en preview o prerelease.
  • La disponibilidad puede depender de región, entorno y licencias.
  • Microsoft 365 Copilot para apps requiere configuración por entorno y app.
Seguridad
  • Copilot hereda permisos, pero el modelo de roles debe revisarse.
  • Column security y datos sensibles pueden limitar experiencias.
  • Las operaciones destructivas requieren especial gobierno y aprobaciones.
Calidad
  • La IA entiende peor tablas con nombres técnicos, relaciones pobres o vistas confusas.
  • Dataverse Search y el semantic model deben tratarse como parte del diseño funcional.
  • Los primeros casos deben tener scope estrecho y datos de alta calidad.
Operación
  • invoke_data_entry presenta sugerencias para revisión; no debe asumirse creación automática sin control.
  • App MCP no elimina la necesidad de procesos deterministas para transacciones críticas.
  • Conviene instrumentar trazabilidad, feedback y criterios de aceptación.
